# Using the Shell

Although the shell is less intuitive than common graphical user interfaces \(GUIs\), most people consider the shell to be much more powerful than a GUI. This is because shells have been around a VERY long time, and many advanced features aren’t available from a graphical desktop environment - they can only be accessed by running shell commands.

## Bourne-Again Shell

The most common shell in use today within Linux is the **B**ourne-**A**gain **SH**ell \(BASH\). The name comes from the shell's compatibility with one of the earliest Unix shells out there, the Bourne shell, named after is creator [Stephen Bourne](https://en.wikipedia.org/wiki/Stephen_R._Bourne).

Whilst bash is included with nearly all modern Linux distributions, other shells are available, such as:

* C Shell \(csh\) - Popular amongst BSD Unix users
* Korn Shell \(ksh\) - Popular among Unix System V users \(and presumably, [KoRn](https://www.kornofficial.com/thenothing/?ref=https://www.google.co.uk/) fans...\)
* Z Shell \(zsh\) - Recently adopted as the default shell in some distributions, such as Kali, as well as MacOS

There are many other shells out there, but this list contains the ones you are most likely to run into.
